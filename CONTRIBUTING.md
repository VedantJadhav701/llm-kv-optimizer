# Contributing to LLM KV Optimizer

First off, thank you for considering contributing to LLM KV Optimizer! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## 🌈 How Can I Contribute?

### Reporting Bugs
*   **Check the Issues**: Before opening a new issue, please check if the bug has already been reported.
*   **Be Specific**: Use the Bug Report template and provide as much detail as possible (logs, hardware specs, steps to reproduce).

### Suggesting Enhancements
*   We love new ideas for KV cache compression!
*   Explain why the enhancement would be useful and how it aligns with the goal of 4GB VRAM optimization.

### Code Contributions
1.  **Fork the repo** and create your branch from `main`.
2.  **Add your logic**: Ensure your code follows the existing style (see `InferenceEngine` for patterns).
3.  **Test your changes**: Run a benchmark using `benchmarks/run_benchmarks.py`.
4.  **Submit a Pull Request**: Describe your changes in detail.

## ⚖️ Code of Conduct
By participating in this project, you agree to abide by the standard Contributor Covenant. Please be respectful and professional in all interactions.

If you're looking for things to work on, check the **Roadmap** section in the README!
